11ty tutorial uit de les gevolgt. (https://11ty.rocks/posts/create-your-first-basic-11ty-website/)

Ik vond de setup van het 11ty project best ingewikkeld om live te krijgen met de github actions.

De voordelen van het makkelijk updaten van de content via markdown files kan ook zonder 11ty want dit heb ik eerder ook al gedaan met mijn i-love-web met liquid. Ik vind nujucks ook niet heel fijn werken.

Nunjucks heeft wel dit: https://www.11ty.dev/docs/data-frontmatter/ wat niet in Liquid kan.

Ik denk dat 11ty best handig kan zijn voor blog sites of simpele websites maar voor onze projecten/ andere use cases zou ik toch eerder svelte + sveltekit gebruiken.

Ik ben zelf sowieso niet heel erg van markdown files gebruiken als database en die stylen. Ik heb liever dat ik json data fetch en zelf bepaal wat voor html alles is ipv dat de automatisch gebeurt.

Het opbouwen van html van html in base.njk is een redelijk hetzelfde als in andere frameworks.
